## 商业化策略概述

### 核心理念

采用 **Freemium（免费增值）** 模式，通过免费版让用户充分体验产品核心价值，建立使用习惯后，通过功能限制和增值服务引导用户升级付费。

### 设计原则

1. **免费版够用** - 覆盖基础学习需求，让用户感受到产品价值
2. **付费有价值** - 高级功能解决深度学习者的真实痛点
3. **转化自然** - 在用户需要时温和提示，不打扰正常使用
4. **限制合理** - 限制点选择在"想要更多"而非"无法使用"

### 目标用户画像

| 用户类型 | 特征 | 适合版本 |
|---------|------|---------|
| 轻度学习者 | 偶尔浏览外文网页，简单学习 | 免费版 |
| 日常学习者 | 每天学习，需要词汇管理 | 免费版 → 高级版 |
| 深度学习者 | 系统化学习，多语言需求 | 高级版 |
| 专业用户 | 翻译工作者、语言教师 | 高级版 |

## 转化策略设计

### 触发点设计

| 用户行为 | 触发时机 | 提示文案 |
|---------|---------|---------|
| 翻译比例调到31%+ | 滑动滑块时 | "升级高级版，解锁更高学习强度" |
| 选择第4种样式 | 点击样式时 | "学习模式是高级功能，升级体验" |
| 收藏第101个单词 | 点击收藏时 | "词库已满，升级享无限收藏" |
| 每日翻译达100段 | 翻译完成时 | "今日额度已用完，明天继续或升级" |
| 尝试使用白名单 | 点击功能时 | "白名单是高级功能" |
| 点击AI词义 | 悬停时 | "AI词义解释是高级功能" |
| 选择非主流语言 | 选择语言时 | "该语言需要高级版支持" |
| 复习第21个单词 | 复习时 | "今日复习额度已用完" |

### 提示原则

1. **温和不打扰** - 只在用户主动触发时提示，不弹窗打断
2. **说明价值** - 告诉用户升级能获得什么，而非只说"不能用"
3. **提供选择** - 允许用户关闭提示或稍后再说
4. **记录频率** - 同一提示24小时内最多显示1次

### 升级入口设计

1. **Popup页面** - 显示当前版本状态，提供升级按钮
2. **Options页面** - 账户设置中显示会员状态
3. **功能限制处** - 被限制功能旁显示"升级"标签
4. **管理后台** - 仪表板显示会员状态和升级入口

## 数据模型设计

### 订阅计划表 (SubscriptionPlan)

```prisma
model SubscriptionPlan {
  id          String   @id @default(uuid())
  name        String   // free, premium
  displayName String   @map("display_name")
  price       Decimal  @db.Decimal(10, 2)
  duration    Int      // 天数，0=终身
  features    Json     // 功能配置JSON
  isActive    Boolean  @default(true) @map("is_active")
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")

  subscriptions Subscription[]

  @@map("subscription_plans")
}
```

### 用户订阅表 (Subscription)

```prisma
model Subscription {
  id        String    @id @default(uuid())
  userId    String    @map("user_id")
  planId    String    @map("plan_id")
  status    String    @default("active") // active, expired, cancelled
  startDate DateTime  @map("start_date")
  endDate   DateTime? @map("end_date") // null=终身
  createdAt DateTime  @default(now()) @map("created_at")
  updatedAt DateTime  @updatedAt @map("updated_at")

  user User             @relation(fields: [userId], references: [id], onDelete: Cascade)
  plan SubscriptionPlan @relation(fields: [planId], references: [id])

  @@index([userId])
  @@index([status])
  @@map("subscriptions")
}
```

### 使用量记录表 (UsageRecord)

```prisma
model UsageRecord {
  id        String   @id @default(uuid())
  userId    String   @map("user_id")
  type      String   // translation, review, collection
  count     Int      @default(0)
  date      DateTime @db.Date
  createdAt DateTime @default(now()) @map("created_at")
  updatedAt DateTime @updatedAt @map("updated_at")

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([userId, type, date])
  @@index([userId])
  @@index([date])
  @@map("usage_records")
}
```
